 # Project Timeline – Medicine Reminder App

## Phase 1 – Project Initiation (Week 1)

- Conduct Project Kickoff Meeting.
- Define project goal: build a Medicine Reminder App.
- Assign team roles and responsibilities.
- Create shared project folder and GitHub repository.

## Phase 2 – Requirement Analysis (Week 2-3)

- Product Manager prepares complete feature list and requirement document.
- Team reviews and confirms core and advanced features.
- Identify technical tools and technologies (Flutter, SQLite, Firebase).

## Phase 3 – UI/UX Design (Week 4-5)

- UX/UI Designer creates wireframes and screen flow in Figma.
- Screens include Login, Home Dashboard, Add Medicine, History, Map, and Settings.
- Team reviews designs and suggests improvements.

## Phase 4 – Core Development (Weeks 6–8)

- Developers implement main features:

  * Add Medicine
  * Set Reminder Time
  * Notification Reminder
  * Medicine History
* Integrate local database (SQLite).
* Conduct basic internal testing.

## Phase 5 – Advanced Feature Development (Weeks 9-12)

- Implement additional features:

  * Edit and Delete Medicine
  * Search and Filter Medicines
  * Dark Mode / Light Mode
  * Map / Pharmacy Locator
  * User Settings

## Phase 6 – Testing and Bug Fixing (Week 13-15)

- Tester performs functional testing and usability testing.
- Identify and track bugs.
- Developers fix reported issues.

## Phase 7 – Final Review and Deployment Preparation (Week 16)

- Final system testing and performance checks.
- Prepare project documentation and presentation.
- Deploy application (prototype/demo version).
